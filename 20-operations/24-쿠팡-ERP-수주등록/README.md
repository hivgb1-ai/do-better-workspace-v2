# 쿠팡 로켓/프레시 ERP 수주등록 자동화

공유 드라이브의 쿠팡 로켓/프레시 발주(OTOO) 파일을 읽어서 사내 ERP "위탁수주 일괄입력" 양식에 자동으로 채워 넣는 스크립트.

## 흐름

1. 대상 날짜(기본: 실행일 기준 전일)의 발주서 폴더를 찾는다.
   - 로켓: `발주서\직매입\쿠팡 로켓\{연도}\{YYMMDD}\`
   - 프레시: `발주서\직매입\쿠팡 프레시\{연도}\{YY.MM.DD}\`
   - 폴더가 없으면(그 채널은 그날 출고 없음) 건너뜀
2. 그 폴더 안에서 파일명에 "OTOO"가 들어간 파일(들)을 찾는다 (본 파일 + "추가"/"재고부족" 등 부가 파일). 그 외 파일(거래명세서 PDF 등)은 무시.
3. 각 파일의 데이터 행에서 거래처코드/품목코드/품목명/주문요청수량/주문금액만 추출.
4. ERP 위탁수주 파일(`ERP 데이터\위탁수주\{YY.MM}\직매입\수주일괄입력_위탁수주_부가세 별도({M월} {로켓|프레시}).xlsx`)에 그 날짜 이름의 새 시트를 만들어 아래 매핑대로 채운다.

## 컬럼 매핑

| 컬럼 | 이름 | 값 |
|---|---|---|
| A~D, K~O, Q~T, W | (고정값) | SOT06 / 1000 / 00 / 00 / egnis / 1000 / 1000 / 1000 / 4660 / KRW / 1 / EA / 1 / 1 |
| G | 판매거래처코드 | OTOO 파일의 거래처코드 |
| P | 수주일 | 대상 날짜 (YYYYMMDD) |
| U | 품목코드 | OTOO 파일의 품목코드 |
| V | 품목명 | OTOO 파일의 품목명 |
| Y | 품목수량 | OTOO 파일의 주문요청수량 |
| **AJ** | 판매금액 | OTOO 파일의 **주문금액** |
| AK, AL, AM | 세무분류/세무코드/세무순번 | 고정값 B / 11 / 1 |
| **AN** | 납품요청일 | P(수주일)과 동일 |

| **BE** | 배송요청일 | P(수주일)과 동일 |
| 나머지 전부 | | 공란 |

위 매핑은 2026-07 기존 완료 시트(로켓 701/706/708/710/713, 프레시 0707/0710/0714)와 대조해서 100% 일치 검증됨.

## 사용법

- **`1_확인만.bat`** — dry-run. 무엇을 할지(대상 파일, 행 수, 첫/마지막 행 미리보기)만 보여주고 아무것도 안 씀. 매일 먼저 이걸로 확인.
- **`2_실제입력.bat`** — 실제로 ERP 파일에 새 시트를 추가. 대상 시트가 이미 있으면 절대 덮어쓰지 않고 건너뜀. 대상 파일이 엑셀에서 열려 있으면(잠금파일 `~$` 존재) 쓰기를 거부.
- 특정 날짜를 지정하려면: `py -3 fill_otoo_to_erp.py --date 2026-07-14` (검증/재실행용)

## 안전장치

- dry-run이 기본값 — `--write` 플래그 없이는 절대 파일을 안 건드림.
- 시트 이름이 이미 존재하면 스킵 (중복 등록 방지).
- 대상 파일이 열려 있으면 저장 거부.

## 시트 이름 규칙 (기존 파일 관찰로 확인됨 — 두 채널이 서로 다름)

- 로켓: `M+DD` (월 앞자리 0 없음, 예: 7월 13일 → `713`)
- 프레시: `MM+DD` (둘 다 0 채움, 예: 7월 14일 → `0714`)

## 검증 이력

- 2026-07-15: `--date 2026-07-14`로 dry-run 실행 → 프레시 950행(OTOO 900 + OTOO 추가 50) 전부 실제 `0714` 시트 값과 100% 일치 확인. 로켓은 해당일 발주 폴더 자체가 없어 스킵(정상).

## 취소 절차 (사용자 제공, 2026-07-15 — 아직 미실행/미검증)

"판매처명 취소해줘" 같은 요청 시 따라갈 절차. **출고처리 여부에 따라 분기.**

### 출고처리가 아직 안 된 상태라면

1. (출고처리 여부 확인 — 안 됐으면 아래로)
2. `일괄납품지시취소입력` 메뉴 조회 > 전일자 기준 조회 > 납품처 지정 조회 > 전체 선택 > 저장
3. `수주처리` 메뉴 조회 > 전일자 기준 조회 > 판매처 지정 조회 > 상태값: **진행**으로 조회 > 전체 선택 > 우측 상단 "확정취소" 클릭
4. `수주처리` 메뉴 조회 > 전일자 기준 조회 > 판매처 지정 조회 > 상태값: **예정**으로 조회 > 전체 선택 > 우측 상단 "수주취소" 클릭

### 출고처리가 이미 된 상태라면

1. `출고취소` 메뉴 조회 > 전일자 기준 조회 > 납품처 지정 조회 > 전체선택 > 우측 상단 "출고취소" 클릭

**주의**: 프로덕션 ERP 데이터를 되돌리는 작업이라 실제 실행은 매번 사용자의 명시적 확정 후에만 진행. 아직 실제로 실행/검증해본 적 없음 — 처음 실행할 땐 결과를 사람이 대조 확인할 것.
