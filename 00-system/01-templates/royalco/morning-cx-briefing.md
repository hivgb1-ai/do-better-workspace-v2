# 아침 CX 현황 브리핑 템플릿

> 로얄앤컴퍼니 CX본부장이 매일 아침 컴퓨터 켰을 때 한 화면에서 확인하는 브리핑.
> 매일 같은 양식으로 나오게 박아두는 용도. 출력은 한 화면(스크롤 없이) 안.

## 입력 데이터

| 파일 | 역할 |
|---|---|
| `complaints.csv` | 사외불만 접수 현황 (최근 30일, 진행단계·심각도·담당부서 포함) |
| `directives.csv` | 본부장이 받은 보고·지시 항목 (이번 주, 핵심 수치·전월 비교 포함) |
| `sales_weekly.csv` | 주간 매출 (불만 ↔ 매출 영향 교차 확인용. 선택 사용) |

## 분석 윈도

- **오늘 기준**: 보고 시점 (월요일 아침 가정)
- 사외불만은 진행 중인 건만 (stage ≠ "종결대기" 그리고 next_action이 있는 건)
- 본부장 지시 항목은 status가 "미확인" 또는 "검토중"인 건만

## 브리핑 출력 구조 (3섹션, 한 화면)

### 1. 한 줄 요약 (헤더)
```
[YYYY-MM-DD 월요일 아침] 즉시 조치 N건 / 오늘 중 확인 M건 / 이번 주 회의 안건 K건
```

### 2. 즉시 조치 필요 (상단, 최대 5건)
컬럼: 건명 / 심각도 / 담당부서 / 마지막 업데이트 / 다음 조치 / 본부장 액션
- 사외불만 중 심각도 "긴급" + 마지막 업데이트 3일 이상 정체된 건 우선
- 사외불만 중 next_action 비어 있는 건 (대책 미수립) 함께 표시
- 본부장 지시 항목 중 status "미확인" + 핵심 수치 전월 대비 악화 (delta < -20%) 건
- 5건 넘으면 심각도 + 정체일 기준으로 5건만

### 3. 오늘 중 확인 (중단, 최대 7건)
컬럼: 건명 / 유형 / 담당부서 / 협의 필요 부서 / 마감일 / 비고
- 사외불만 중 심각도 "중대" + 3~7일 정체된 건
- 본부장 지시 항목 중 status "검토중" + 마감일 오늘~내일
- 협의 필요 부서가 있는 건은 별도 표시 (어떤 부서와 협의해야 하는지)

### 4. 이번 주 진행 모니터링 (하단, 표만, 최대 5건)
컬럼: 건명 / 심각도 / 진행단계 / 담당부서 / 마지막 업데이트
- 위 1·2에 들어가지 않은 진행 중 건 중 심각도 "중대" 이상

## 검출 규칙 (이상 신호)

| 신호 | 기준 | 색상 |
|---|---|---|
| 긴급 클레임 정체 | severity = "긴급" + (오늘 - last_update) ≥ 3일 | 빨강 |
| 대책 미수립 | next_action 비어 있음 | 빨강 |
| 동일 batch 의심 | title에 "동시" "다발" "batch" 포함 | 빨강 |
| 중대 클레임 정체 | severity = "중대" + (오늘 - last_update) ≥ 5일 | 노랑 |
| VIP·관급·건설사 클레임 | channel = "건설사" 또는 title에 "VIP/관급" 포함 | 노랑 |
| 본부장 지시 핵심 수치 악화 | this_value vs prev_value delta < -20% | 빨강 |
| 본부장 지시 핵심 수치 급등 (부정) | 불량률·미응답률·연체율 등 +50% 이상 | 빨강 |
| 협의 필요 부서 미확정 | coord_dept_needed 비어 있고 dept 단독 처리 어려운 건 | 노랑 |
| SNS 부정 확산 위험 | title에 "SNS/후기" 포함 | 노랑 |

## 길이·톤 제약

- **한 화면 안** (모니터 스크롤 없이 전체 보임). 텍스트는 표 + 짧은 헤더만.
- 표 셀 안 줄바꿈 금지. 마지막 업데이트는 날짜 또는 "N일 전" 형식.
- 톤: 사실·정체일·다음 조치 중심. 본부장이 5분 안에 오늘의 우선순위 잡을 수 있게.
- 추측·해석 금지. "현장조사 결과 필요" 같은 사실 표기.
- 같은 건이 두 섹션에 중복 등장 금지.

## 산출물 형식

기본: 마크다운(.md). 본부장이 추가 질문 던지면 같은 데이터 안에서 답변. 한 줄 요청으로 PDF용 HTML(로얄앤코 디자인) 변환 가능.

## 주의

- **이종미 본부장이 브리핑 받은 직후 추가 질문을 던지는 흐름을 가정한다.** "이 RB1530 클레임들 같은 batch야?" 같은 즉문에 동일 데이터 안에서 답할 수 있어야 한다.
- "즉시 조치"와 "오늘 중 확인"의 경계는 정체일 + 심각도로 결정. 임의 판단 금지.
- 종결대기 또는 next_action이 있고 정체 3일 미만인 건은 브리핑에서 제외 — 본부장이 신경 쓸 필요 없음.
- 동일 제품에 클레임이 5건 이상 몰리면 별도 알림 박스로 분리 ("batch 의심").
