## 상황 설정

**월요일 아침 8시 40분. 화성 본사 영업총괄 자리.**

밖은 5월 햇살이 벌써 따갑다. 책상에는 금요일 밤 전산에서 내려받은 분기 자료 네 개가 그대로 열려 있다 — `sales_weekly.csv`, `customers.csv`, `special_requests.csv`, `receivables.csv`. 오전 10시에 분기 영업정책 보고가 잡혀 있다. **대표님 보고**. 머리 한쪽에 회장님이 지난 분기 회의 마무리에 던지셨던 한마디가 떠다닌다. "약정 미달 영업사원, 환원유예로 덮지 말고 가져와."

슬랙 알림이 울린다.

> **박소영 이사님**: 부장님, 10시 보고 자료 미리 톤 좀 맞춰주실 수 있을까요
>
> 저희 문화사업부 직영점 3곳도 이번 분기 약정에 들어가 있어서요

> 대표님 스타일 아시잖아요

> 그러고는 "그 사람 거래처에서 환원유예 들고 있는 거 몇 개야?" 바로 따라옵니다
> 환원유예 케이스가 표 위에서 분리돼 있어야 답이 됩니다

**(속으로)**: 대표님... 분기마다 같은 질문이다. 영업사원 이름 → 거래처 → 환원유예 보유 여부 → 미달 이유. 표 따로 만들어 가져가면 "그래서 이 사람 어떻게 할 거야?" 한마디로 끝난다. 의사결정 안건이 표 위에 명확하게 와 있어야 9시 30분에 박소영 이사한테 톤 공유하고 10시 보고에 들어갈 수 있다.

슬랙이 또 온다.

> **방인철 부장님**: 부장님, 한 가지만요
> 지난주 RUE420 부속 단종 건 때문에 수배부족이 12개 품번까지 늘었습니다

**(속으로)**: 특별출고와 부속 부족이 겹치면 대표님이 "왜 부속도 없는데 특별출고를 승인했어?" 하실 게 뻔하다. 그것까지 같이 표 위에 안 올라가면 미팅에서 두 부서가 서로 떠넘기는 그림이 된다.

- 실제 가용 시간: 지금부터 50분
- 받은 파일: sales_weekly.csv, customers.csv, special_requests.csv, receivables.csv

**Claude Code가 없었다면?**
- 14주 매출 데이터 영업사원별로 합계 + 약정 대비 이행률 계산 (50분)

**Claude Code가 있다면?**

- 박소영 이사 요청(직영점 분리) + 방인철 부장 요청(부속 부족 ↔ 특별출고 교차) 추가 질문 (15분)

## 미션 1: 분기 영업정책 이행 보고서 1차 산출

대표님이 가장 먼저 물어보실 부분이다. 약정 미달 영업사원·채널·거래처 → 환원유예 분리.

**(속으로)**: 대표님이 표 펴자마자 손가락으로 짚으실 거다. "이 영업사원, 환원유예 몇 개?" 그러면 옆 표에서 바로 보여야 한다. 분석 템플릿에 검출 규칙(이행률 < 85% 노랑·< 70% 빨강·원가배수 < 1.45 빨강·환원유예 보유 + 이행률 < 80% 빨강)이 다 박혀 있으니 한 번에 호출하자. 채널 5개 (전문점·대리점·건설사·직영점·수출) + 영업사원 10명 + 특별요청 38건 + 매출채권 53곳 — 손으로 합치면 보고가 시작된다.

**분석 요청**:
```
@sales_weekly.csv @customers.csv @special_requests.csv @receivables.csv
@templates/royalco/quarterly-sales-policy-report.md 관점으로
이번 분기(직전 13주) 영업정책 이행 보고서 만들어줘.
채널 5개 + 영업사원 10명 + 특별요청 검토 알림 + 의사결정 후보 3건까지.
대표님 분기 보고용이라 환원유예 보유 거래처는 일반 미달과 분리해서 표시해줘.
```

**결과 확인 후 추가 질문 예시**:
```
"박소영 이사 요청 — 직영점 채널 3곳을 별도 표로 빼서, 거래처별 분기 약정 대비
이행률 + 매출채권 잔액까지 같이 보여줘. 정산 보고에서 그대로 쓸 수 있게."
```

```
"방인철 부장 요청 — 이번 주 특별출고 승인 건 중 RUE420 부속 부족 거래처와 겹치는
건이 있는지 확인해줘. 거래처명 + 영업사원 + 사유 + 상태로."
```
