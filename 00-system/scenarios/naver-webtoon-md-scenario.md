# 데이터 분석 실습 시나리오 - 네이버웹툰 IP 머천다이징

> **타깃 업무**: 웹툰 IP 기반 굿즈 유통/스토어 운영, 라이선시 발주, 외부 채널 정산
> **샘플 데이터**: `50-resources/sample-data/naver-webtoon-md/`
> **팀**: 네이버웹툰 IP 사업실 라이선싱&머천다이징 비즈니스 팀

> ⚠️ **허구 고지**: 등장 인물(동료·팀장·라이선시 담당자), 거래처명(영글미디어·쓰리스타즈·YL캐릭터), 이메일 주소는 **교육용 허구**이며 실제 네이버웹툰 인력·거래처와 무관. 이메일 도메인은 RFC 2606의 `.example` 예약 TLD 사용. 팀명·IP명·스마트스토어/이카운트 CSV 포맷·수수료 체계·리드타임 패턴 등 **업무 구조와 데이터 포맷은 현실 기반**.

## Mission 1: 주간 발주 판단 + 라이선시 발주 메일 초안

> **유진 팀장님**: 은원님, 주말 주문 많았죠?
> 발주 판단 좀 서둘러 주세요
>
> 판매종료 검토할 품목도 같이 봐주세요
> 3개월간 5개 미만 나간 건 솔직히 재고 더 들일 이유가 없어요

> 라이선시별로 나눠서 정리해주시면 좋겠어요

**(속으로)**: 한 번에 가면 더 헷갈리니까 단계별로 가자. 일단 최근 30일 주문 전체를 상품/옵션 단위로 피벗부터 뽑자.


## Mission 2: 카카오 + 29CM 월간 정산 집계 + 로열티 산출

민아님과 준호님의 요청. 월초 정산 루틴.

> **민아님**: 카카오 선물하기 3월분이랑 29CM 3월분 정리해서
> 작품별 로열티까지 떨어진 표 좀 부탁드려요
>
> 채널별로 포맷이 다른 게 문제에요
> 카카오는 한글 컬럼인데 29CM는 영문이거든요
> 작품명도 약간씩 다르게 표기되어 들어올 때가 있어요

> **준호님**: 29CM는 할인가로 팔린 게 많아서
> 실제 수수료 차감 후 정산금이 얼마인지 정확히 봐야 해요

**(속으로)**: 두 채널 포맷이 다르지만 결국 "작품 × 굿즈유형 × 판매금액" 단위로 정리되면 같은 틀이다. 마스터 파일로 묶으면 된다.

### 분석 요청 (단계 1 — 채널별 작품 단위 집계)

```
@naver_webtoon_kakaogift_sales_2603.csv 와 @naver_webtoon_29cm_sales_2603.csv 를 같이 봐줘.

두 파일이 포맷이 달라 (카카오는 한글 컬럼, 29CM는 영문).
먼저 공통 스키마로 정리해줘: 주문일 | 채널 | 상품코드 | 상품명 | 옵션 | 수량 | 판매단가 | 판매금액

그다음 @naver_webtoon_products_master.csv 를 조인해서
작품명(IP)을 붙이고, 작품 × 채널별로 판매수량·판매금액 합계를 피벗해줘.
```

### 결과 확인 후 후속 질문

```
@naver_webtoon_channel_fees.csv 기준으로 채널별 수수료를 차감해 정산금을 계산해줘.
- 카카오 선물하기: 기본 10% + 결제수수료 3.3%
- 29CM: 기본 25% + 결제수수료 3.0%
- 수수료 차감 후 정산금 열을 추가

그다음 @naver_webtoon_ip_royalty.csv 의 작품별 로열티율을 적용해
로열티 금액도 계산해줘. 로열티는 "수수료 차감 후 정산금" 기준이야.

최종 열 구성:
채널 | 작품명 | 판매수량 | 판매금액 | 채널수수료 | 정산금 | 로열티율 | 로열티금액 | 순수익
```

```
29CM 에서 할인율 영향이 얼마나 있는지 따로 봐줘.
- 정가(ListPrice) 기준 예상매출 vs 실제 판매가(SalePrice) 기준 실매출
- 작품별로 할인 영향 금액 차이
- 준호님이 "정산금이 예상보다 적다"고 한 게 할인 때문인지 수수료율 때문인지 판단할 수 있게
```

```
구글 스프레드시트에 붙여넣을 포맷으로 마지막 정리 부탁해.
- 시트1: 채널×작품 교차표 (행: 작품, 열: 채널, 값: 정산금)
- 시트2: 로열티 지급 예정 표 (라이선시별 합계까지 내려가게)
- 시트3: 원시 데이터 (채널·주문일·작품·판매금액·수수료·정산금·로열티)

TSV로 뽑아주면 그대로 복붙할게.
```

## 데이터 간 연결고리

```
                    products_master (SKU, IP코드, 작품명, 라이선시)
                           │
              ┌────────────┼────────────────────┬────────────┐
              │            │                    │            │
      smartstore_orders    │             kakaogift_sales  29cm_sales
         (상품명으로)      │              (상품코드=SKU)  (ItemNo=SKU)
              │            │                    │            │
              ▼            ▼                    ▼            ▼
         [판매 집계]   [일평균]            [채널별 매출]
              │            │                    │
              │            │                    ▼
              │            │             channel_fees
              │            │             (수수료 차감)
              │            │                    │
              │            │                    ▼
              │            │             ip_royalty
              │            │             (로열티 계산)
              │            │
              ▼            ▼
      ecount_inventory (품목명 불일치 18%)
              │
              ▼
      leadtime_licensee (IP × 굿즈 × 라이선시)
              │
              ▼
      [발주 필요 수량 + 라이선시별 메일 초안]
```

**핵심 교차 지점**:
- `SKU` — products_master × 카카오/29CM 매출 (코드 매칭)
- `상품명` — products_master × smartstore/ERP (텍스트 매칭, 불일치 규칙 필요)
- `IP코드` — products_master × ip_royalty × leadtime_licensee
- `라이선시` — leadtime × ip_royalty (발주 메일 수신자 / 로열티 정산 대상)
