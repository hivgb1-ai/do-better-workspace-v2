## 매일 아침 한 화면 보기

본부장·임원·점장 누구든 아침에 어제 무슨 일 있었는지 한 번에 보고 싶을 때. 회의 들어가기 전 5분.

### 그냥 이렇게 말해보세요

```
어제 운영에서 큰 일 있었는지 한 화면으로 보여줘.
어제는 2026-02-16 기준으로 잡고,
매출·AS 접수·카드 입금 정도 같이 보면 좋겠어.
```

처음엔 표 하나가 나옵니다. 그걸 보고 떠오르는 대로 또 시키면 됩니다.

```
8일 넘게 정체된 AS 건이 따로 보였으면 좋겠어. 어느 거래처가 가장 많은지도.
```

```
입금이 평소보다 늦은 카드사·매장이 있으면 색깔이라도 다르게.
```

### 매일 다른 결과가 나오는 게 핵심

같은 부탁을 날짜만 바꿔서 다시 시켜보세요. 1월 첫 영업일, 1월 중순, 2월 시작, 2/16, 2/22, 3월 마감 푸시 시점.

**같은 도구·다른 신호.** 어떤 날은 평온, 어떤 날은 정체된 건이 갑자기 늘어나 있고, 어떤 날은 카드 입금이 이상하게 밀려 있습니다. 회의에서 "오늘 뭐 봐야 해?"라는 질문에 답할 수 있는 표가 매일 다르게 나옵니다.

### 어떤 사람이 쓰면 좋은가

- CX 본부장 — 어제 클레임 추세, 정체된 건, SNS 반응 같이 보고 임원회의 입장
- 매장 운영 책임자 — 매장별 입금·매출 한눈에 + 카드사 지연 잡아내기
- 경영진 — 운영 전체가 정상인지 5분 안에 확인

## 매주 월요일 영업이 어떤지 보기

영업총괄·임원이 매주 같은 표를 보면서 "이번 주 누가 흔들리고 있나"를 잡고 싶을 때.

### 그냥 이렇게 말해보세요

```
이번 주(8주차) 영업이 어떤지 표 한 장으로 정리해줘.
누가 약정 잘 가고 있고 누가 미달인지,
환원유예 신청한 사람들 + 특별출고 많이 받은 사람도 같이.
```

표가 나오면, 거기서 또 떠오르는 대로.

```
약정 미달인 사람들의 거래처 중에 매출채권 90일 넘은 곳이 있는지 같이 보여줘.
```

```
특별출고 횟수가 평소 대비 너무 많은 사람이 있으면 그 사람만 따로 표시.
```

### 주차를 바꿔서 또 돌려보세요

3주차·6주차·9주차·12주차. 처음엔 다들 비슷해 보이지만, 누적이 쌓이면서 누가 진짜 흔들리는지 점점 드러납니다. **"5주차엔 안 보이던 게 9주차엔 보인다"**는 경험이 이 도구의 핵심 가치.

### 어떤 사람이 쓰면 좋은가

- 영업총괄 부장 — 매주 월요일 정확히 같은 보고
- 경영진 — 분기 약정 추세 + 환원유예 누가 들고 있는지
- 인사·재무 — 매출채권 위험 거래처 사전 식별

## 매월 초 정산하기 전에 확인하기

위탁업체 정산·카드 입금·매출 데이터 사이에 미스매치가 있는지 한 번에 확인하고 싶을 때.

### 그냥 이렇게 말해보세요

```
2월 정산하기 전에 좀 봐줘.
매출 합계랑 위탁업체 정산 합계가 안 맞는 게 있나,
같은 품번인데 단가가 다르게 적용된 거래가 있나,
입금이 예상보다 적게 들어온 매장이 있나.
```

표가 나오면 거기서 자연스럽게.

```
어떤 거래처가 가장 미스매치 많아? 그 거래처의 기안번호별로 차이금액 보여줘.
```

```
품번 이름이 바뀐 거(WC2xxx-N) 중에 신품번으로 안 잡힌 거래가 있어?
```

```
○○건설(주)·○○건설·○○建設처럼 같은 곳인데 표기가 다른 현장을 하나로 묶어줘.
```

### 월을 바꿔서 또 돌려보세요

1월·2월·3월. 1월엔 거의 깨끗합니다. 2월부터 단가 변경 미스매치가 보이기 시작하고, 3월엔 시즌 마감 푸시까지 겹쳐서 정산 검증할 게 많아집니다.

### 어떤 사람이 쓰면 좋은가

- 재무회계·원가 담당 — 수불부 + 계정 매칭 자동화의 시작점
- 이지포스 정산 담당 — 매출-재고-정산 3엑셀 크로스체크
- CX/AS 분석 — 품번 변경 이력과 현장명 보정

## 한 번 만들어두면 매번 다시 쓸 수 있게

위 3가지를 한 번씩 해보고 "이거 매일/매주/매월 쓸 수 있겠다"는 감이 오면 그대로 시키면 됩니다.

```
지금까지 한 흐름을 '매일 아침 운영 보기' 스킬로 저장해줘.
다음에는 그냥 "오늘 아침 보기" 한마디면 같은 표가 나오게.
```

```
이 흐름을 '주간 영업 점검' 스킬로 저장. 주차만 바꿔서 부르면 표 새로 그려지게.
```

```
'월 정산 검증' 스킬로 저장. 다음 달부터 매월 1일 아침에 같은 형식으로.
```

스킬로 저장한 다음에는 잘못 나온 부분만 그때그때 알려주면 됩니다. "이 경우엔 이렇게 처리해" 한마디로 스킬이 업데이트됩니다.

## 작게 시작해서 깊이 들어가기

처음부터 위의 큰 질문 하나를 던지지 마세요. 작은 것부터 한 단계씩.

```
1단계: "이 엑셀에 시트가 몇 개 있어? 각각 뭐야?"
2단계: "as_received 시트에서 어제 접수된 건 몇 개야?"
3단계: "그중에 8일 넘게 안 끝난 건 따로 보여줘."
4단계: "어느 거래처가 그 정체 건을 가장 많이 가지고 있어?"
5단계: "그 거래처의 어제 매출도 같이 보여줘."
6단계: "이걸 한 화면으로 합쳐서 매일 볼 수 있게 만들어줘."
```

**한 번에 모든 걸 시키지 마세요.** 하나 보고 → 다음 떠오르는 것 → 또 다음. 결과 보고 떠오르는 게 자연스럽게 다음 질문이 됩니다. 이게 평소에 본인이 일하던 방식이고, AI한테도 같은 방식이 통합니다.

### 데이터 품질 패턴 (자연스럽게 발견되는 함정)

- `sales_daily`에 월 합계 행 3개 자연 삽입
- 위생도기 7품번(WC2107·2114·2121·2128·2135·2142·2149)이 2월부터 `-N` 표기, 구품번 컬럼에 변경 이력
- `as_received` 현장 5곳 표기 흔들림 (`○○건설`/`○○건설(주)`/`○○建設`)
- `products` 정수기 일부 카테고리 흔들림 (`정수기`/`정수기류`/`정수`)
- 직영 채널 행은 영업사원 칸 빈 셀, B2B 5%도 영업사원 누락
- `dealer_settlement` 승인여부 일부 빈 셀

### 세 루틴이 누적되며 드러나는 큰 그림

- 매일 아침 루틴의 BC카드 지연 매장(C·E·H) → 매주 점검의 특정 영업사원 담당 매장과 매핑
- 매주 점검의 환원유예 마스킹 영업사원_03·07 → 매월 정산의 매출채권 90일+ 거래처와 80% 겹침
- 매일 루틴의 RB1530 batch → 매월 정산의 단가 변경 시점과 무관한 별개 라인 → 하지만 같은 거래처 묶음(E·F·K·L·P)에서 동시 발생

**AI는 이 연결을 한 번에 못 찾습니다.** 참가자가 세 루틴을 각자 2~3회 돌린 뒤 본인이 "어? 이 거래처 또 나오네"라고 발견하는 순간이 인사이트의 시작.

### 강도 의도

- RB1530·단가변경: 중간 강도 (1차 분석에서 보일 수 있음)
- 환원유예·BC카드·시즌: 약하게 (1~3주차엔 안 보이고 5주차부터 누적)
