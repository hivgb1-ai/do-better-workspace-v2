# ledger-reconcile — 장부 맞추기 (공통 뼈대)

정형 입구의 표 데이터(거래내역·발행 기록)를 맞춰 **미수금·미발행을 자동 감지**한다. 이 스킬은 절차(엔진)만 안다 — 분류 패턴·거래처·임계값 같은 **내 값은 전부 내 세계 사전에 산다**.

**읽는 파일 (판단 재료는 전부 파일에 — 이 스킬 본문에 개인 데이터 없음):**

```bash
WS_ROOT=$(git rev-parse --show-toplevel 2>/dev/null || pwd)
```

- **선언표** `$WS_ROOT/00-system/08-registry/자리-선언표.yaml` — `정형입구` 절(입력 파일 위치·신선도기대) + `표면.장부`(갱신 대상. 없으면 첫 실행 때 생성 제안)
- **사전** `$WS_ROOT/00-system/08-registry/내-세계-사전.yaml` `장부` 절 — 자동분류 패턴·본인 이체 패턴·age 임계값 (비어 있으면 전부 unclassified로 시작 — 정상)

## 절대 규칙 (엔진 — 누구에게나 같음)

1. **파일도 원본이다** — 입력 파일(xlsx·CSV)은 쌓기만 하고 고치지 않는다. 원장(ledger)에는 무중복 append만 (중복 키: 날짜+시각+금액+잔액 류의 원본 필드 조합).
2. **추측 분류 금지** — 사전 패턴에 확신이 없으면 `unclassified`로 남긴다. 미분류 0건이 목표지만, 도달 못해도 거짓 분류보다 낫다.
3. **장부 재생성 금지 (기존 메모 보존)** — 사람이 적은 비고·맥락은 보존하고 age·금액·매칭 결과만 Edit로 갱신 (Write 전체 덮어쓰기 금지).
4. **금전 상태 확정은 사실 근거로만** — "입금 완료"는 거래내역에서 해당 입금을 직접 확인했을 때만. 발행 ≠ 입금.
5. **todo 등록은 후보 제시까지** — 오래된 미수금의 확인 todo는 후보로 차려 사용자 확정 후 append (실행 수위 ②).

## Step 1: import — 정형 입구 → 원장

- 선언표 `정형입구`에 선언된 위치에서 최신 파일 감지 (또는 인자로 받은 파일)
- 원장(ledger CSV)에 신규 행만 append — 중복 키로 이미 있는 행은 건너뜀
- 컬럼 구조: 원본 컬럼 그대로 + 라벨 컬럼(분류·거래처·메모) 추가

## Step 2: 자동 분류

사전 `장부.자동분류-패턴`의 우선순위대로 각 행을 분류:

1. 본인 이체 패턴 → 내부 이체 (매칭 대상 아님)
2. 고정비 패턴 → 고정비
3. 거래처 별칭 매칭 → 해당 거래처
4. 그 외 → `unclassified` (사용자 확인 요청)

첫 실행에서 unclassified가 많은 게 정상 — 사용자가 분류를 확정할 때마다 그 패턴을 사전에 등록해 다음부터 자동화한다 (쓸수록 똑똑해지는 지점).

## Step 3: 매칭 — 발행 vs 입금

거래처별로 발행 합계와 입금 합계를 대조해 판정:

| 판정 | 뜻 | 장부 반영 |
|------|-----|----------|
| 일치 | 발행 = 입금 | 정상 — 장부에 안 올림 |
| 분할 | 나눠서 입금 중 | 정상 — 비고에 분할 표시 |
| 미수 | 발행 있는데 입금 0 | **§미수금** (발행일 기준 age 계산) |
| 선입금 | 입금 있는데 발행 0 | **§미발행** |

## Step 4: 장부 갱신

- 선언표 `표면.장부` 파일을 Edit로 행 단위 갱신 (규칙 3 — 메모 보존)
- age 컬럼 갱신: 오늘 − 발행일/입금일. 사전 `장부.age-임계값` 초과 건에 경고 마커
- 상단 "마지막 업데이트" 날짜 갱신 (morning 도착신선도 점검이 이 날짜를 본다)

## Step 5: 보고 + todo 후보

```
import: 원장 +N행 / 분류: 자동 M · unclassified K
매칭: 일치 N / §미수금 N건 (최장 age NN일) / §미발행 N건
제안 todo: "[거래처] 입금 확인 (발행 D+NN)" — 추가할까요? [y/n]
```
