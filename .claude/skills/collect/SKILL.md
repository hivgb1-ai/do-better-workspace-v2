# /collect — 수집 러너

선언표에 등록된 수집기 중 **돌릴 수 있는 것을 돌리고, 상태 파일이 갱신됐는지 확인한다. 그게 전부다.**

수집은 흐름(자동·수시)이고 아침 점검은 리듬이다 — 그래서 morning과 분리돼 있다.
이 스킬 파일에는 채널·경로·명령이 없다. **무엇을 돌릴지는 전부 자리 선언표가 안다.**

## 누가 부르나

1. **크론/자동화** — 주기 실행
2. **사람** — 아무 때나 ("수집 한 바퀴 돌려줘")
3. **morning 이어가기** — 바퀴 1에서 실행 신선도 초과 경고가 떴을 때

## 수행 절차

### 1. 선언표 읽기

```bash
WS_ROOT=$(git rev-parse --show-toplevel 2>/dev/null || pwd)
```

`$WS_ROOT/00-system/08-registry/자리-선언표.yaml`의 **수집기** 절을 Read.

### 2. 실행 대상 선별

수집기마다:
- **실행명령이 선언돼 있으면** → 실행 대상.
- 실행명령이 없으면 → 건너뜀 + 이유 한 줄 (방식이 `손`이면 "손 수집 채널", `조회형`이면 "실행 대상 아님", 합류 대기 중이면 선언표의 해당 계획 필드를 인용).
- 인자로 특정 수집기 이름을 받으면 그것만 돌린다.

### 3. 실행

각 대상의 `실행명령`을 그대로 실행한다 (`$WS_ROOT`는 위에서 정한 값으로 치환).
실패해도 다른 수집기는 계속 돌린다 — 수집기 하나의 실패가 바퀴를 멈추지 않는다.

### 4. 갱신 확인 (약속 2 검사)

실행한 수집기마다 선언표의 `상태파일`을 확인:
- 상태 파일의 마지막 시각(last_updated 류)이 **실행 후 시각으로 갱신됐는가**
- 갱신 안 됐으면 실패로 본다 — 수집기가 돌았다는 주장과 남긴 흔적이 달라선 안 된다

### 5. 보고

```
=== /collect (YYYY-MM-DD HH:MM) ===
| 수집기 | 실행 | 상태파일 갱신 | 새 수집 |
|--------|------|---------------|---------|
| ...    | OK/실패/건너뜀(이유) | 확인/미갱신 | N건 |
```

- 새 수집분 개수는 수집기 출력 또는 원본함(선언표 `표면.원본함`) 신규 파일로 센다.
- **실패 시**: 에러 원문 1~2줄 + 선언표 그 수집기의 `복구` 필드 안내를 보여준다. 원인 미상의 실패를 조용히 넘기지 않는다.

## 하지 않는 것

- **분류** — raw에 떨구기까지만. 가르기는 inbox-triage.
- **신선도 감시·경고** — morning 바퀴 1의 일. (여기서는 "방금 돌린 것"의 갱신 확인만.)
- **표면 수정** — 원본함에 쌓기만 (원본은 쌓기만 한다).
