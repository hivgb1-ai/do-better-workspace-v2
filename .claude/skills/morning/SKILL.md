# morning — 선언표를 도는 아침

아침 상태를 감지하고 상태판을 출력한다. **감지·노출은 read-only** — 자동교정은 점검 선언이 명시적으로 허용하고 사실 확정 근거가 있는 경우에만.

이 스킬은 자기 점검 목록을 갖지 않는다. **점검 항목·기준값·개인 항목은 전부 선언표와 사전에 산다** — 새 채널·새 점검은 선언표에 한 줄 등록하면 자동으로 감시에 들어온다 (스킬 수술 없음).

## 읽는 파일

```bash
WS_ROOT=$(git rev-parse --show-toplevel 2>/dev/null || pwd)
TODAY=$(date +%Y-%m-%d); MONTH=$(date +%Y-%m)
DAY_NUM=$(date +%u)
case $DAY_NUM in 1) WEEKDAY="월";; 2) WEEKDAY="화";; 3) WEEKDAY="수";; 4) WEEKDAY="목";; 5) WEEKDAY="금";; 6) WEEKDAY="토";; 7) WEEKDAY="일";; esac
```

- **선언표**: `$WS_ROOT/00-system/08-registry/자리-선언표.yaml` — 수집기·아침점검·이어가기·개인점검
- **사전**: `$WS_ROOT/00-system/08-registry/내-세계-사전.yaml` — 선언표가 `값출처:`로 가리키는 값만

## 바퀴 0 — 동기화 (유일한 고정 항목)

```bash
cd "$WS_ROOT" && git fetch --dry-run 2>&1; git status -sb | head -1
```
behind 있으면 `[todo]` git pull, 없으면 `[done]`. (원격이 없으면 조용히 통과.)

## 바퀴 1 — 수집기 신선도 (read-only)

선언표 **수집기** 절의 각 항목에서, 선언된 신선도만 본다. **수집 실행은 하지 않는다** — 실행은 /collect의 일.

| 신선도 | 묻는 것 | 보는 곳 | 잡는 고장 |
|--------|---------|---------|-----------|
| **실행 신선도** | 수집기가 마지막으로 언제 돌았나 | 선언된 `상태파일`의 마지막 시각 | 수집기 자체 멈춤 (크론 죽음·스크립트 에러) |
| **도착 신선도** | 최신 항목이 언제 것인가 | 원본함 raw의 선언된 날짜 필드 최신값 | 수집기는 도는데 **원천이 죽음** (전달앱 사망 등) |

- `실행신선도`/`도착신선도`가 선언된 수집기만 본다 (`없음`·`[빈칸]`은 건너뜀).
- 기대치 초과 → `[todo]` 경고. 실행 신선도 초과면 **/collect 이어가기 후보**, 도착 신선도 초과면 원천 생존 점검을 강조.
- 상태 파일이 없으면 "기록 없음 = 이상"으로 표시 (없음을 정상으로 착각하지 않기).

도착 신선도 범용 스니펫 (날짜 필드명·기대일은 선언표에서):

```bash
# find 기반 — 원본함이 비어도 zsh "no matches found" 에러가 안 난다 (glob 대신)
find "$WS_ROOT/00-inbox/raw" -maxdepth 1 -name '*.md' -exec \
  grep -hoE "^(kakao_date|source_date): *[0-9]{4}-[0-9]{2}-[0-9]{2}" {} + 2>/dev/null \
  | grep -oE "[0-9]{4}-[0-9]{2}-[0-9]{2}" | sort | tail -1
```

## 바퀴 2 — 표면 점검

선언표 **전체**(표면·파일밖표면·수집기 어디든)에서 `아침점검:` 선언을 모아 실행한다. 유형은 5종뿐이고, 유형별 실행 방법은 아래가 전부다. 값(위치·패턴·한계·키워드)은 선언에서 온다.

| 유형 | 실행 방법 |
|------|-----------|
| **실행신선도** | 바퀴 1과 동일 (수집기 쪽 선언) |
| **도착신선도** | 선언된 날짜패턴을 grep으로 추출 → 오늘과의 차이가 `기대` 초과면 선언된 `알림` |
| **개수** | `위치`+패턴이면 grep 카운트 / `명령`이면 그 명령 실행 후 출력 해석. 알림 기준에 따라 `[todo]`/`[skip]`/정보 표시. 명령 에러는 `[warn]`으로 노출 (숨기지 않음) |
| **헤더룰** | 파일 헤더 블록에서 선언된 날짜패턴을 모두 추출, `한계` 초과 건수 카운트 |
| **달력키워드** | 내일 일정을 캘린더 연동으로 뽑아 `값출처`가 가리키는 사전의 포함/제외 키워드로 매치 → 매치 시 사전의 안내문 표시. 캘린더 연동이 없으면 이 유형은 건너뛴다 |

범용 스니펫:

```bash
# 헤더룰 — 날짜 비교는 awk (zsh [ \< ]는 "condition expected" 에러)
if [ "$(uname)" = "Darwin" ]; then CUTOFF=$(date -v-${LIMIT_DAYS}d +%Y-%m-%d); else CUTOFF=$(date -d "${LIMIT_DAYS} days ago" +%Y-%m-%d); fi
head -40 "$FILE" | grep "^> " | grep -oE "$DATE_PATTERN" | grep -oE "20[0-9]{2}-[0-9]{2}-[0-9]{2}" \
  | awk -v c="$CUTOFF" '$0 < c' | wc -l | tr -d ' '
```

**유형 규율**: 유형을 늘리지 않는다. 여섯 번째 유형이 필요해 보이면 일단 개인 점검(자유 서술)으로 두고, 여러 사람에게 반복될 때만 유형으로 승격한다. (모델이 세질수록 과잉 뼈대가 해가 된다 — 그 경계를 스킬 안에 새겨두는 것.)

## 바퀴 3 — 이어가기

선언표 **이어가기** 절: 조건이 참이면 선언된 순서대로 해당 스킬을 잇는다. morning은 시동만 — 제안·정리는 이어진 스킬이, **밖으로 나가는 것의 확정은 언제나 사람이**.

- 각 항목의 `조건`은 바퀴 0~2에서 이미 잰 값으로 판정한다 (재측정 없음).
- `상세`의 지침(덮어쓰기 금지·먼저 알릴 것·순서)을 그대로 따른다.
- **상태판을 먼저 다 보여준 뒤** 이어간다 — 아침 상태판을 후속 작업이 가리지 않게.
- 이어간 스킬이 길어지면 사용자가 중간에 끊어도 된다.

## 개인 점검

선언표 **개인점검** 절의 각 항목을 `상세`(자유 서술)대로 실행한다. 사람마다 자기 것을 등록하는 영역. 자동교정은 개인점검 선언이 명시적으로 허용한 것만, 선언된 사실 확정 절차를 거쳐서.

## 출력 — 상태판

```
Morning Routine - {TODAY} ({WEEKDAY})

[상태] 1. {항목}   -- {한 줄 설명}
[상태] 2. ...
...

다음: /{첫 번째 todo 또는 next 항목}
```

- 항목 순서: 바퀴 0 → 바퀴 1(수집기별) → 바퀴 2(아침점검 선언 순) → 개인점검 → 이어가기 예고. 번호는 이 순서에서 동적으로.
- 조용히 통과하는 항목(선언에 "0건이면 표시 생략" 류)은 선언대로.

| 상태 | 의미 |
|------|------|
| `[done]` | 이미 완료됨 |
| `[next]` | 다음으로 실행할 스킬 (1개만) |
| `[todo]` | 해야 할 일 |
| `[skip]` | 건너뛰기 (데이터 없음) |
| `[warn]` | 점검 자체가 에러 (숨기지 않고 노출) |

**규칙**: `[next]`는 첫 번째 미완료 항목에만 부여. 나머지 미완료는 `[todo]`.
